简历Pro - 产品功能规格说明书
=====================================
最后更新：2026年4月29日
=====================================


一、产品定位
=====================================
产品名称：简历Pro
产品类型：AI驱动的在线简历生成工具
核心理念：让每个普通人都能写出一份专业的简历
目标用户：中国大陆应届生、职场新人（22-30岁）

核心使用场景：
1. 用户打开网页 → 直接使用，无需注册登录
2. 粘贴目标岗位JD → AI分析匹配度
3. 引导式填写简历 → 每一步都有AI辅助
4. 导出PDF → 投递简历


二、技术栈
=====================================
前端：Next.js 14 (App Router) + TypeScript + Tailwind CSS + shadcn/ui
后端：Next.js API Routes
数据库：Supabase（免费额度）
AI：DeepSeek API（deepseek-chat模型）
部署：Vercel（计划中）
状态管理：Zustand（persist中间件，数据存localStorage）


三、数据模型（数据库表结构）
=====================================

表1：users（用户表）
- id: string (uuid)
- email: string
- name: string
- avatar: string (可选)
- plan: 'free' | 'monthly' | 'yearly'
- created_at: timestamp

表2：resumes（简历表）
- id: string (uuid)
- user_id: string (外键 → users.id)
- title: string
- slug: string
- profile: jsonb (基本信息和个人简介)
- work: jsonb[] (工作经历数组)
- education: jsonb[] (教育背景数组)
- skills: jsonb[] (技能特长数组)
- projects: jsonb[] (项目经历数组)
- certifications: jsonb[] (证书数组)
- languages: jsonb[] (语言能力数组)
- achievements: jsonb[] (成就/奖项数组)
- interests: string[] (兴趣爱好)
- references: jsonb[] (推荐人)
- created_at: timestamp
- updated_at: timestamp

表3：templates（模板表）
- id: string
- name: string
- slug: string
- thumbnail: string (预览图URL)
- category: 'simple' | 'professional' | 'creative' | 'modern'
- is_premium: boolean

表4：orders（订单表）
- id: string
- user_id: string
- plan: string
- amount: number
- status: 'pending' | 'paid' | 'failed' | 'refunded'
- paid_at: timestamp

表5：ai_versions（AI改版记录）
- id: string
- resume_id: string
- tone: 'default' | 'foreign' | 'startup'
- content: jsonb
- created_at: timestamp


四、页面结构与功能
=====================================

【4.1 首页 - JD智能匹配】
--------------------------------------
入口：打开网址 http://localhost:3000 默认显示

功能清单：
- 顶部标题"简历Pro" + 副标题"粘贴JD，30秒知道简历匹配度"
- 粘贴JD文本框（placeholder有示例）
- 清空按钮
- "开始分析"按钮
- 分析结果区域：
  * 环形进度条 + 分数（0-100分）
  * 匹配评价标签（"匹配度很高"/"匹配度中等"/"匹配度偏低"）
  * 缺失关键词建议（蓝色标签，可点击）
  * 修改建议列表
  * "开始制作简历"按钮
- "生成简历体检报告"卡片（分享功能）
- 底部版权信息

当前AI逻辑（待改进）：
- 提取关键词：techScore×0.5 + softScore×0.3 + quantScore×0.2
- 问题：词库不分岗位，字符串匹配，非语义理解


【4.2 简历编辑器 - 主界面】
--------------------------------------
进入方式：首页点击"开始制作简历"按钮

顶部工具栏（sticky）：
- 左侧：Logo"简历Pro"
- 中间：模板选择下拉框
- 右侧按钮组：
  * ✨ AI优化（全模块AI优化，显示结果浮层）
  * "返回JD匹配"按钮
  * "导出PDF"按钮（触发浏览器打印）

左侧表单区（固定宽度420px）：
- 顶部Tab切换栏（sticky）：
  [基本信息] [个人简介] [工作经历] [教育背景] [技能特长] [项目经历]
  当前选中Tab：蓝色高亮+下划线

- 表单内容区（可滚动）：

【基本信息】
- 专业选择：下拉框，选专业后显示推荐岗位标签（12-15个/专业）
- 姓名（必填）
- 求职目标（必填，可手动输入或点击推荐标签填充）
- 邮箱
- 手机
- 所在城市
- 个人网站/博客
- 码云/Gitee
- 保存提示：字段变化后显示"✅ 已自动保存"（2秒后消失）
- 自动保存：Zustand persist，开关页面不丢失

【个人简介】
- "看范例"按钮：展开3个优秀简介参考（技术岗/运营岗/产品岗）
- "引导编写"按钮：弹出3步引导
  * 步骤1：选定位（6选1：技术达人/项目实践者/学习能力强/综合发展/数据驱动/产品思维）
  * 步骤2：回答3个问题（最擅长的技能/最骄傲的成就/3年目标）
  * 步骤3：预览生成的简介，可手动编辑
- 文本输入框（placeholder提示公式：身份背景+核心能力+代表成就+职业目标）
- "✨ AI优化简介"按钮：调用API增量优化+结构压缩
- AI优化结果：显示优化后内容，用户可选择"使用这个"或"继续修改"

【工作经历】
- "✨ 引导式添加"按钮：弹出3步引导
  * 步骤1：填基本信息（公司/职位/开始时间/结束时间）
  * 步骤2：选择工作类型（多选：开发功能/优化改善/主导项目/协作项目/解决难题/数据分析）
  * 步骤3：STAR法则描述
    - S（背景）：✨AI帮写按钮 + 文本框
    - T（任务）：✨AI帮写按钮 + 文本框
    - A（行动）：✨AI帮写按钮 + 文本框
    - R（结果）：✨AI帮写按钮 + 快捷量化提示 + 文本框
    - 技术栈：手动输入框（回车添加）+ 已添加标签可删除
  * 保存按钮
- 已有工作经历列表：
  * 每条显示：公司名/职位/时间 + ✨AI优化按钮 + 删除按钮
  * 展开显示STAR内容
- "再添加一段经历"按钮
- AI帮写逻辑：
  * 根据用户选的工作类型，AI生成对应场景的参考描述
  * 用户可点选参考句直接填入，或与已有内容合并

【教育背景】
- "✨ 引导式添加"按钮：弹出3步引导
  * 步骤1：基本信息（学校名称/学校层次6选1/专业/时间）
  * 步骤2：成绩与荣誉（GPA/排名/荣誉奖项多选）
  * 步骤3：预览保存
- 已有列表：显示学校/专业/学历/时间/GPA/荣誉标签
- 荣誉字段：保存在education[].achievements数组

【技能特长】
- "✨ 引导式添加"按钮：弹出2步引导
  * 步骤1：选择领域（16个领域：前端/后端/移动端/AI/数据/DevOps/数据库/产品/运营/金融/语言/软技能等）
  * 步骤2：选择技能（每个领域有预设技能列表）+ 熟练程度下拉（精通/熟练/熟悉/了解）
    + 手动输入框：自定义技能名称（回车添加）
- 已添加分类：显示技能标签+星级（★）
- 熟练程度：保存在skills[].skills[].level（1-5）

【项目经历】
- "✨ 引导式添加"按钮：弹出3步引导
  * 步骤1：基本信息（项目名称/项目类型5选1/角色）
  * 步骤2：技术栈（手动输入框，回车添加）
  * 步骤3：STAR描述
    - S（背景）：✨AI帮写 + 文本框
    - T（任务/角色）：✨AI帮写 + 文本框
    - A（行动）：✨AI帮写 + 文本框
    - R（结果）：✨AI帮写 + 快捷量化提示 + 文本框
  * 项目链接（可选）
  * 保存按钮

右侧预览区（实时渲染）：
- A4纸张比例（max-w-[210mm]）
- 根据选中的模板渲染
- 数据来源：Zustand store（currentResume）
- 所有模板切换实时生效


【4.3 模板系统 - 5套模板】
--------------------------------------
template-1（默认）：技术岗 - 左侧深色栏+右侧内容（黑白）
template-2：金融/咨询 - 传统严谨（白底）
template-3：快消/运营 - 活泼现代（橙色渐变）
template-4：国企/体制内 - 正式传统（表格化头部）
template-5：创意/设计 - 视觉突出（双栏布局）

每个模板渲染的字段：
- 姓名/求职目标/联系方式
- 个人简介（profile.summary）
- 工作经历（work[].company/position/highlights[]）
- 教育背景（education[].institution/degree/field/gpa/achievements[]）
- 技能特长（skills[].category/skills[].name/level）
- 项目经历（projects[].name/role/technologies[]/highlights[]）
- 技术栈显示在对应模块中


五、AI能力详情
=====================================

【5.1 AI优化类型】
--------------------------------------
| type参数   | 功能           | 输入                      | 输出                     |
|------------|----------------|---------------------------|--------------------------|
| summary    | 简介优化       | 现有内容+目标岗位          | 增量优化+结构压缩        |
| star_s     | STAR-S生成     | 工作类型+场景              | 1-2个背景参考句         |
| star_t     | STAR-T生成     | 工作类型+岗位              | 1-2个任务参考句         |
| star_a     | STAR-A生成     | 工作类型+技术栈            | 1-2个行动参考句         |
| star_r     | STAR-R生成     | 工作类型+行业              | 1-2个量化结果参考        |
| skills     | 技能应用场景   | 技能名+岗位               | 应用场景描述             |
| project    | 项目描述优化   | 项目内容                   | 优化后的描述             |
| education  | 教育背景优化   | 学校+专业+GPA             | 优化后描述               |
| full       | 全模块优化     | 完整简历                   | 所有模块优化+评分+建议   |

【5.2 全模块优化返回格式】
--------------------------------------
{
  "optimizedSummary": "优化后的个人简介",
  "optimizedWork": [...],        // 优化后的工作经历
  "optimizedSkills": [...],      // 优化后的技能
  "optimizedProjects": [...],     // 优化后的项目
  "optimizedEducation": "...",    // 优化后的教育描述
  "overallScore": 0-100,         // 整体评分
  "overallSuggestions": [...],    // 整体建议（3-5条）
  "moduleScores": {               // 各模块评分
    "summary": 0-100,
    "work": 0-100,
    "skills": 0-100,
    "projects": 0-100,
    "education": 0-100
  }
}

【5.3 AI调用方式】
--------------------------------------
接口地址：POST /api/ai-generate
请求体：
{
  "resume": Resume对象,
  "targetRole": "前端开发工程师",
  "type": "summary|star_s|star_t|star_a|star_r|skills|project|education|full",
  "workItem": {...},    // 可选，指定某条工作经历
  "extra": {...}        // 可选，额外参数
}

返回格式：
{
  "success": true|false,
  "data": {...},         // AI返回的JSON数据
  "error": "错误信息"     // 出错时返回
}

【9.3 我们的优势】
--------------------------------------
1. 无门槛使用：打开即用，不需要注册登录
2. 引导式填写全流程：STAR法则贯穿每一步
3. 专业→岗位智能映射：选专业自动推荐目标岗位
4. AI全流程辅助：从基本信息到每个工作经历都有AI帮写
5. 专为中国人设计：中文优化、中文岗位库、国内模板风格
6. DeepSeek成本低：比GPT/Claude便宜50%+

【9.4 我们的劣势/不足】
--------------------------------------
1. 品牌认知度为零：用户不知道我们，没有信任背书
2. 没有人工点评：超级简历有HR背书，用户更信任
3. 没有登录系统：无法保存简历、无法做历史版本
4. 没有多版本改写：竞品简历牛有外企版/大厂版/创业版
5. 没有分享主页：无法生成个性化链接分享简历
6. 没有移动端：竞品都有App或移动端适配
7. 没有社交证明：没有用户评价、案例展示


十、自对标分析 - 不足之处与改进建议
=====================================

【10.1 产品层面不足】
--------------------------------------

不足1：品牌信任感缺失
→ 问题：用户看到新网站第一反应是"这靠谱吗"
→ 改进建议：

  * 可以考虑接入微信登录，利用微信背书

不足2：没有人工背书
→ 问题：超级简历有HR人工点评，用户更信任
→ 改进建议：
  * 短期：展示真实用户评价截图

十四、API接口详细文档
=====================================

【14.1 AI生成接口】
--------------------------------------
POST /api/ai-generate

请求头：
Content-Type: application/json

请求体参数：
{
  "resume": Resume对象,       // 必填，当前简历完整数据
  "targetRole": string,       // 可选，目标岗位
  "type": string,            // 必填，优化类型
  "workItem": object,         // 可选，指定工作经历
  "extra": object            // 可选，额外参数
}

type可选值：
- "summary": 个人简介优化
- "star_s": STAR-S生成
- "star_t": STAR-T生成
- "star_a": STAR-A生成
- "star_r": STAR-R生成
- "skills": 技能应用场景
- "project": 项目优化
- "education": 教育背景优化
- "full": 全模块优化

成功响应：
{
  "success": true,
  "data": {...}  // 具体返回格式见5.2节
}

失败响应：
{
  "success": false,
  "error": "错误描述"
}
